[S-4]: FDO Record Root Profile Validation

Input: List of FDO attributes a1=(a1,def,a1,val ),..., an=(an,def,an,val ).

Process: Check that there are at least the attributes `0.FDO/Type`, `0.FDO/Profile` and `0.FDO/Data` in the a1,def,...,an,def.

Output: An integer

  • 0 (COMPLETE; all three attributes were found)
  • 1
  • 2
  • 3
IMPORTANT: It seems that all the steps are generic. Hence, writing down how the building blocks are represented in the implementation seems enough to enable processing. However, maybe it makes sense to add some more details in the FDO spec on where the implementation specifications come into play (e.g., attribute mappings, etc).
