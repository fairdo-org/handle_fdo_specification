[S-3] PID Record Parsing into an FDO Record

Input: JSON

Process: Map all handle values from the JSON to FDO attributes as described in [H3-2].

Some of the attributes in the HandleResponse are related to the functioning of the Handle system. A client MAY heuristically exclude them from processing since they are clearly distinguishable from the FDO-related attributes.
Beware: Excluding such attributes on modification of a Handle record can have severe consequences! For example, removing the `HS_ADMIN` attribute causes the record to be never editable again.
There exists no official list of such Handle-related attributes, but an incomplete list can be found here:
  • HS_* (everything starting with HS; e.g., HS_ADMIN, HS_SITE)
  • 0* (Handle root records can be ignored; except 0.FDO prefix)
  • 10320/* (e.g., 10320/loc)
function parseFDORecord (json: HandleResponse) -> FDORecord throws HandleRecord {
    const pid = json["handle"]; // Get PID from JSON
    let attributes: {
        key : string,
        value : string
    }[] = [];

    for row in json["values"]{ // Iterate over every row in the Handle record
        const key: string | undefined = row["type"] || undefined;
        const value: string | undefined = row["data"]["value"] || undefined;

        if (!key || !value) {
            throw new HandleError("No key or value in JSON", row);
        } else {
            attributes.append(key, value);
        )
    }

    return new FDORecord(pid, attributes);
}

Output: List of attributes a1=(a1,def,a1,val ),..., an=(an,def,an,val ). The ai,def can either be attribute definition PIDs or attribute names.
