Inline-FDOs

[H6-1] Representation of Inline-FDOs
An inline-FDOs is represented as a JSON Object. The keys are either the PID of the attribute definitions or human-readable names. The values are always an array of all the attribute values in the FDO record. This is needed to avoid conflicts resulting from possibly multiple attribute-value tuples from the FDO record that use the same attribute definition. Every inline-FDO must at least comply with the root profile, other profiles may be added as needed. Using profile referencing, an inline-FDO can also be mandated to comply with other profiles.

As an example, we assume an FDO record that describes a dataset and includes an inline-FDO that describes a file included in the dataset. The FDO record could look similar to this:

PID of Attribute Definition Example Value
21.T11969/Profile 21.T11969/Root
21.T11969/Type 21.example/dataset
21.T11969/Data 21.example/file
21.example/hasPart 
{
    "21.T11969/Profile": ["21.T11969/Root", "21.example/dataset"],
    "21.T11969/Type": ["21.example/fileFormat"],
    "21.T11969/Data: ["21.example/dataset"],
    "21.example/fileFormat": ["image/jpeg"],
    "21.example/HTTPURL": ["https://test.example/image01.jpg"],
    "21.example/fileSizeInKB": ["4242"],
    "21.T11969/Name": ["Placeholder01"],
    "21.example/isPartOf": ["21.example/dataset"]
}
21.example/hasPart {...} (another inline FDO)
21.example/hasPart 21.example/existingFile01 (profile reference to an already existing FDO)
