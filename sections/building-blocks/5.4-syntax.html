[H5-4] Syntax
There are two ways how to encode syntactical validation rules in the Handle record:
  • Way 1: The FDO record specifies a JSON schema which contains all validation rules through the attribute 21.T11969/Data. This attribute applies attribute referencing to point to the attribute 21.T11969/JsonSchema, which has as its value a JSON schema. The JSON Schema encodes the following five basic validation rules:
    1. An attribute states to follow one of the primitive data types “string”, “number”, “integer” and “boolean” using the `type` keyword.
    2. An attribute states to comply with a regular expression using the `pattern` keyword.
    3. An attribute states to lie within a numeric range using the keywords `minimum`, `exclusiveMinimum`, `maximum` or `exclusiveMaximum`.
    4. An attribute states to be equal to one element from a whitelist through the `enum` keyword.
    5. An attribute states to be not equal to all elements from a blacklist through combining `not` with `enum`.
    In the first example, the JSON schema is directly written into the FDO record of the FDO attribute definition.
    PID of Attribute Definition Example Value
    21.T11969/Type FDO_Attribute
    21.T11969/Profile 21.T11969/AttributeDef
    21.T11969/Name publication year
    21.T11969/Description The publication year of the resource.
    21.T11969/Cardinality 1
    21.T11969/Data 21.T11969/JsonSchema
    21.T11969/JsonSchema 
    {
    "$schema": "http://json-schema.org/draft-04/schema#",
    "title": "publication year",
    "description": "The publication year of the resource.",
    "type": "integer",
    "minimum": 0,
    "maximum": 2026
}
    Profile FDO record (fictionary PID: 21.example/attribute01)
    In the second example, the JSON schema is stored somewhere else and is referenced in the FDO record.
    PID of Attribute Definition Example Value
    21.T11969/Type FDO_Attribute
    21.T11969/Profile 21.T11969/AttributeDef
    21.T11969/Name color
    21.T11969/Description The color of the resource.
    21.T11969/Cardinality 0..*
    21.T11969/Data 21.T11969/JsonSchema
    21.T11969/JsonSchema 
    {"$ref": "https://typeapi.lab.pidconsortium.net/v1/types/schema/21.example/attribute02"}
    Profile FDO record (fictionary PID: 21.example/attribute02)
    The JSON schema can be found under the URL https://typeapi.lab.pidconsortium.net/v1/types/schema/21.example/attribute02: 
    {
    "$schema": "http://json-schema.org/draft-04/schema#",
    "title": "color",
    "description": "The color of the resource.",
    "type": "string",
    "enum": ["red", "yellow", "green", "blue"],
    "not": {
        "enum": ["black", "white"]
    }
}
  • Way 2: The FDO attribute definition is fully encoded on the Handle record level, simply relying on a reference to a syntax definition. To specify that way 2 is used to encode the syntax of an attribute, the following two attributes must be used in the FDO record of the attribute definition:
    1. (21.T11969/ValidationMechanism, Syntax)
    2. (21.T11969/DataType, [reference to a syntax definition]).
    Syntax Definitions are themselves FDOs and encode the five basic validation rules in their records. They follow the syntax definition profile with the PID 21.T11969/SyntaxDef and contain the attributes listed in Table 5.
    Table 5: The list of attribute definitions specified by the FDO syntax definition profile (21.T11969/SyntaxDef). These attributes must be used when defining any new syntax definition. This table does not include all information about the attributes. Please resolve the attribute definition PIDs via the Handle System.
    PID of attribute definition Description of the attribute Cardinality of the attribute
    21.T11969/Type The string FDO_Syntax_Definition. Additional types may be added, if applicable. 1..*
    21.T11969/Profile The string 21.T11969/SyntaxDef. Additional profiles may be added, if applicable. 1..*
    21.T11969/Data The string Not_Applicable. 1..*
    21.T11969/Name Human-readable and localized labels for this syntax. 1..*
    21.T11969/Description Human-readable and localized descriptions for this syntax. 0..*
    21.T11969/PrimitiveDataType string, number, integer, or boolean 1
    21.T11969/Regex regular expression in ECMA-262 syntax 0..1
    21.T11969/NumericInterval An interval using one of the following forms: [a,b], [a,b), (a,b], (a,b). 0..1
    21.T11969/Whitelist Any instance of the primitive data type. If 21.T11969/Whitelist appears multiple times, this gives an exhaustive list of allowed values for this attribute. 0..*
    21.T11969/Blacklist Any instance of the primitive data type. If 21.T11969/Blacklist appears multiple times, this gives a non-exhaustive list of forbidden values for the attribute. 0..*

  • Note that both ways can be combined by using all attributes 21.T11969/Data, 21.T11969/JsonSchema, 21.T11969/DataType, and 21.T11969/ValidationMechanism simultaneously. In this case, when creating or updating this FDO attribute definition, it must be checked that both ways induce the same syntax validation rules.
