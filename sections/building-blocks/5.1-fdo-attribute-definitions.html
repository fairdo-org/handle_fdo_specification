FDO attribute definitions

[H5-1] Representation of FDO attribute definitions
The Handle record of a Handle-based FDO attribute definition must contain a fixed, predefined set of FDO attributes (see Table 4). Note that there are two types of attributes in the Handle record:
  1. Meta information about the attribute definition, to describe the FDO attribute definition itself: 21.T11969/Type, 21.T11969/Profile, 21.T11969/Name (all mandatory), 21.T11969/Description (optional).
  2. The actual content of the attribute definition, from which all validation rules (for attributes instantiating this attribute definition) can be derived: 21.T11969/Cardinality, 21.T11969/Data (both mandatory), 21.T11969/JsonSchema, 21.T11969/ValidationMechanism, 21.T11969/DataType (optional). There are several mechanisms to encode and interpreted attribute definitions. Each of those mechanisms have their own validation rules for instantiating FDOs. The mechanisms and their validation rules are explained in more detail, including examples, in their respective sections:
    • [H5-4] Syntax
    • [H5-5] Compositions of Attribute Definitions
    • [H5-6] Unions of Attribute Definitions
    • [H5-7] Profile Referencing
    • [H5-8] Attribute Referencing
    • [H5-9] External Vocabulary Referencing
    A short summary of how to encode each of these rules in a Handle-based attribute definition is given in Table 4. The full explanations are given in the respective sections [H5-4] - [H5-9].
Table 4: The list of attribute definitions which MUST be used when registering a new Handle-based FDO attribute definition. This table does not include all information about the attribute definitions. For more details, please resolve the attribute definition PIDs via the Handle System.
PID of the attribute definition Attribute value Cardinality of the attribute
21.T11969/Type The string FDO_Attribute_Definition. Additional types may be added, if applicable. 1..*
21.T11969/Profile The string 21.T11969/AttributeDef. Additional profiles may be added, if applicable. 1..*
21.T11969/Name Human-readable and localized label(s) for the attribute definition. 1..*
21.T11969/Description Human-readable and localized description(s) for the attribute definition. 0..*
21.T11969/Cardinality Description of the obligation and repeatability of this attribute in an FDO record. 1
21.T11969/Data This attribute can have two different values:
  • The string 21.T11969/JsonSchema, if this attribute definition defines a syntax/union/composition according to way 1.
  • The string Not_Applicable, if this attribute definition defines syntax/union/composition according to way 2; or if it defines a profile reference / attribute reference / external vocabulary reference.
If both ways are combined for syntax/union/composition, then the value is 21.T11969/JsonSchema. 		1
21.T11969/JsonSchema This attribute is just used when the attribute represents a syntax/union/ composition according to way 1. In this case, the value is an inline JSON schema, representing an FDO attribute definition. 0..*
21.T11969/ValidationMechanism This attribute is just used when the attribute represents a syntax/union/ composition according to way 2, or a profile reference / attribute reference / external vocabulary reference. The value is of:
  • Syntax (see [H5-4])
  • InlineCombination (see [H5-5])
  • Union (see [H5-6])
  • AttributeReference (see [H5-7])
  • ProfileReference (see [H5-8])
  • ExternalVocabularyReference (see [H5-9])
 0..*
21.T11969/DataType This attribute is just used when the attribute represents a syntax/union/ composition according to way 2, or a profile reference / attribute reference / external vocabulary reference. The value is a reference to one of:
  • a syntax definition (if the validation mechanism is Syntax)
  • an attribute definition (if the validation mechanism is Union, InlineCombination, or AttributeReference)
  • a profile (if the validation mechanism is ProfileReference)
  • an external vocabulary namespace (if the validation mechanism is ExternalVocabularyReference)
0..*
