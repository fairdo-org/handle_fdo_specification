[H5-6] Unions of Attribute Definitions

There are two equivalent ways how to encode the union of attribute definitions in Handle-based attribute definitions.

[H5-6a] Unions implemented by JSON Schemas:

The FDO record specifies a JSON schema using the attribute 21.T11969/Data. This attribute applies attribute referencing to point to the attribute21.T11969/JsonSchema, which has as its value a JSON schema encoding a union of attributes:

The JSON schema uses the keyword anyof to compose different attribute definitions. Each attribute definition can be either a reference through the $ref keyword, or it can be directly pasted into the JSON schema as a sub schema. 

{
"$schema": "http://json-schema.org/draft-04/schema#",
"title": "Union of string and integer",
"description": "This attribute validates all strings and all integers.",
"anyOf": [
{
  "$ref": "https://typeapi.lab.pidconsortium.net/v1/types/schema/21.T11969/3df63b7acb0522da685d"
},
{
  "$ref": "https://typeapi.lab.pidconsortium.net/v1/types/schema/21.T11969/9edcffff4461c9083fbf"
}
]
}
This attribute union validates: 
123
and 
"myString"

[H5-6b] Unions implemented on Handle-record level:

  1. (21.T11969/ValidationMechanism, Union)
  2. (21.T11969/DataType, [reference to attribute definition]), repeated multiple times.
The following three attributes define a union of a string and an integer.
  1. (21.T11969/ValidationMechanism, Union),
  2. (21.T11969/DataType, 21.T11969/3df63b7acb0522da685d),
  3. (21.T11969/DataType, 21.T11969/9edcffff4461c9083fbf).
This attribute union validates 
123
and 
"myString"
.
