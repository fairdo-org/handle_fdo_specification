[H5-6] Unions of Attribute Definitions

An attribute definition A is a union of existing attribute definitions A1,...,An if it validates any attribute that is valid according to at least one of A1,...,An. There are two equivalent ways how to build a union of existing attribute definitions as a Handle-based FDO attribute definition.

[H5-6a] Unions implemented by JSON Schemas:

The FDO record of the attribute definition A contains a JSON schema which encodes a union of attributes A1,...,An. In detail, the attribute 21.T11969/Data applies attribute referencing to point to the attribute 21.T11969/JsonSchema, which has as its value a JSON schema. The JSON schema uses the keyword anyof to compose the different attribute definitions A1,...,An into a union. Each attribute definition can be either a reference through the $ref keyword, or it can be directly pasted into the JSON schema as a sub schema.

The following FDO attribute definition record defines a union of integers in the interval [1990,2000] and [2010,2020].
PID of Attribute Definition Example Value
21.T11969/Type FDO_Attribute
21.T11969/Profile 21.T11969/AttributeDef
21.T11969/Name year_1990s_2010s
21.T11969/Description Any year in the 1990s and 2010s.
21.T11969/Cardinality 1
21.T11969/Data 21.T11969/JsonSchema
21.T11969/JsonSchema 
{
    "@id": "hdl:21.T11969/example_integer_1990s",
    "$schema": "http://json-schema.org/draft-04/schema#",
    "title": "year_90s_2010s",
    "description": "Any year in the 1990s and 2010s.",
    "anyOf": [
        {
            "@id": "hdl:21.T11969/example_integer_1990s"
            "title": "year_1990s",
            "description": "Any year in the 1990s.",
            "type": "integer",
            "minimum": 1990,
            "maximum": 1999
        },
        {
          "$ref": "https://typeapi.lab.pidconsortium.net/v1/types/schema/21.T11969/example_integer_2010s"
        }
    ]
}
FDO record of attribute definition (fictionary PID: 21.example/union_attribute01)
The attribute definition validates for example the attributes 
(21.example/union_attribute01, 1993)
and 
(year_1990s_2010s, 2010).

[H5-6b] Unions implemented on Handle-record level:

The FDO record of the attribute definition A must specify the attributes 21.T11969/ValidationMechanism and 21.T11969/DataType to define a union of attributes A1,...,An. In detail, the following attributes must be in the attribute definition:
  • (21.T11969/ValidationMechanism, Union) to state that this attribute definition is a union
  • (21.T11969/DataType, A1), ..., (21.T11969/DataType, An) to state that the union contains the attribute definitions A1,...,An.
The following FDO attribute definition record defines a union of integers in the interval [1990,2000] and [2010,2020].
PID of Attribute Definition Example Value
21.T11969/Type FDO_Attribute
21.T11969/Profile 21.T11969/AttributeDef
21.T11969/Name year_1990s_2010s
21.T11969/Description Any year in the 1990s and 2010s.
21.T11969/Cardinality 1
21.T11969/Data NotApplicable
21.T11969/ValidationMechanism Union
21.T11969/DataType 21.T11969/example_integer_1990s
21.T11969/DataType 21.T11969/example_integer_2010s
FDO record of attribute definition (fictionary PID: 21.example/union_attribute01)
The attribute definition validates for example the attributes 
(21.example/union_attribute01, 1993)
and 
(year_1990s_2010s, 2010).
