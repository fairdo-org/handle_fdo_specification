Attribute mapping

All (conceptual) mandatory and optional attributes from the [[FDO specification]] are implemented as Handles. The attributes from the [[FDO specification]] have the format `0.FDO/[X]` and represent FDO-related concepts. They are mapped to Handle-based FDO attribute definitions by exchanging the prefix 0.FDO with the prefix 21.T11969. The Handle-based attributes are registered in the Data Type Registry. You can resolve the PIDs using the Handle System. A full mapping of the conceptual PIDs from the FDO specification to Handle-based attributes is given in Table 1 and Table 2 for completeness. The third columns give more details on how each attribute is implemented using Handle-based FDOs.

0.FDO/Type 21.T11969/Type The type of the FDO as specified in section 3.10 of the [[FDO specification]]. For Handle-based FDOs, this attribute uses attribute referencing, i.e., the value is a reference to another attribute which gives more information about the type. 1..*
0.FDO/Profile 21.T11969/Profile A PID that resolves to an FDO profile. 1..*
0.FDO/Data 21.T11969/Data The data of the FDO. For Handle-based FDOs, this attribute uses attribute referencing, i.e., the value is a reference to another attribute which gives more information on how the data are linked (e.g., through a PID resolving to another FDO (collection), through a URL pointing to a file, or directly inline in the Handle record).
