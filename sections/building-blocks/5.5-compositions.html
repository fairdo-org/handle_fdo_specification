Composition of Attribute Definitions There are two ways how to encode the composition of attribute definitions in Handle-based attribute definitions:

  • Way 1: A composition of attributes is implemented in JSON schema as an object with properties. Each property corresponds to one attribute definition. The properties must all be arrays, where links to the JSON schema of another attribute definitions by either using the keyword $ref, or the other attribute definition is fully given inline. The composition is instantiated by a JSON that validates against the JSON schema.

    JSON does not support having the same key several times. Hence, the cardinalities of the attribute definitions (given on the Handle record-level by the attribute 21.T11969/Cardinality) that are part of the composition are ignored. To state that certain attributes are required (Cardinality 1), you can use the required keyword. Additionally, you can use the additionalProperties keyword to prevent adding additional content to the composition.

    The following FDO record gives the attribute definition of an author, which is a composition of a first name and a last name.
    PID of Attribute Definition Example Value
    21.T11969/Type FDO_Attribute
    21.T11969/Profile 21.T11969/AttributeDef
    21.T11969/Name author
    21.T11969/Description An author is a composition of a first name and a last name..
    21.T11969/Cardinality 1
    21.T11969/Data 21.T11969/JsonSchema
    21.T11969/JsonSchema 
    {
    "$schema": "http://json-schema.org/draft-04/schema#",
    "title": "author",
    "description": "An author is a composition of a first name and a last name.",
    "type": "object",
    "properties": {
        "firstName": {
            "$ref": "https://typeapi.lab.pidconsortium.net/v1/types/schema/21.T11969/3df63b7acb0522da685d"
        },
        "lastName": {
            "title": "String",
            "description": "Validation schema for type 'String' with the PID '21.T11969/3df63b7acb0522da685d'",
            "type": "string"
            }
        }
    },
    "required": ["firstName", "lastName"],
    "additionalProperties": false
}
    Profile FDO record (fictionary PID: 21.example/comp_attribute01)
    This attribute definition validates the attribute: 
    ("21.example/comp_attribute01", {"firstName": "Jon", "lastName": "Doe"})
  • Way 2: A composition of attributes is implemented by setting two attributes in the FDO record:
    1. (21.T11969/ValidationMechanism, InlineCombination)
    2. (21.T11969/DataType, [reference to attribute definition]), repeated multiple times.
    The following three attributes define a composition of a string and an integer.
    1. (21.T11969/ValidationMechanism, Composition),
    2. (21.T11969/DataType, 21.T11969/3df63b7acb0522da685d),
    3. (21.T11969/DataType, 21.T11969/9edcffff4461c9083fbf),
    The composition is instantiated by the following JSON object (String and Integer are the names of the attributes -> we need to define better examples here): 
    {"String": "abc", "Integer": 123}
