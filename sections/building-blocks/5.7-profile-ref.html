[H5-7] Profile Referencing

Profile referencing permits an attribute definition A to specify that the value v (of an attribute a = (k: v) that instantiates A) MUST be a reference to an FDO following at least one profile out of a set of profiles P1,...,Pn. Hence, profile referencing is a mechanism to enforce that a specific attribute in an FDO record references another FDO that follows a certain profile. There are two equivalent ways how to build a Handle-based attribute definition that applies profile referencing (see [H5-7a] and [H5-7b]).

Validation Rule: To validate an attribute a = (k: v) that applies profile referencing, it is checked that v is either a reference to another FDO following at least one of the profiles P1,...,Pn, or that it is an in-line serialization of an FDO that instantiates at least one of P1,...,Pn. If no profile Pi is specified, it means that v is simply a reference to an FDO, i.e., that the referenced FDO complies to the root profile 21.T11969/Root.

[H5-7a] Profile Referencing implemented by JSON Schemas:

The FDO record of the attribute definition A must contain a JSON schema which encodes a profile reference to one of the profiles P1,...,Pn. In detail, the attribute 21.T11969/Data applies attribute referencing to point to the attribute 21.T11969/JsonSchema, which has as its value a JSON schema. The JSON schema can specify both syntax rules for the attribute value (e.g. that it is a string or follows a regular expression) and validation rules that need to be checked beyond the syntax of the attribute value (profile referencing). The latter rules must be extracted and checked separate from simple JSON schema validaton: 
{
"validation": {
    "mechanism": "profileReference",
    "allowedProfiles": ["[reference to P1], ..., [reference to Pn]"]
}
validation. If the allowedProfiles key is left out, it means that the referenced FDO must be complient with 21.T11969/Root.
The attribute definition 21.T11969/Profile uses profile referencing. An implementation of its FDO record could look like this:
PID of Attribute Definition Example Value
21.T11969/Type FDO_Attribute
21.T11969/Profile 21.T11969/AttributeDef
21.T11969/Name FdoProfile
21.T11969/Description A reference to an FDO profile.
21.T11969/Cardinality 1..n
21.T11969/Data 21.T11969/JsonSchema
21.T11969/JsonSchema 
{
    "$schema": "http://json-schema.org/draft-04/schema#",
    "@id": "hdl:21.T11969/Profile",
    "title": "FdoProfile",
    "description": "A reference to an FDO profile.",
    "type": "string",
    "validation": {
        "mechanism": "profileReference",
        "allowedProfiles": ["21.T11969/ProfileDef"]
    }
}
FDO record of attribute definition (PID: 21.T11969/Profile.)

[H5-7b] Profile Referencing implemented on Handle-record level:

The FDO record of the attribute definition A must specify the attributes 21.T11969/ValidationMechanism and 21.T11969/DataType to define a profile reference to one of the profiles P1,...,Pn. In detail, the following attributes MUST be in the attribute definition:
  • (21.T11969/ValidationMechanism, ProfileReference) to state that this attribute definition uses profile referencing
  • (21.T11969/DataType, P1), ..., (21.T11969/DataType, Pn) to state that the attribute value must point to an FDO that is valid against at least one of the profiles P1, ..., Pn. If 21.T11969/DataType is not contained in the attribute definition, it means that the referenced entity is an FDO (hence, that the value references an FDO complying to at least 21.T11969/Root).
The attribute definition 21.T11969/Profile uses profile referencing. Its FDO record looks as follows:
PID of Attribute Definition Example Value
21.T11969/Type FDO_Attribute
21.T11969/Profile 21.T11969/AttributeDef
21.T11969/Name FdoProfile
21.T11969/Description A reference to an FDO profile.
21.T11969/Cardinality 1..n
21.T11969/Data Not_Applicable
21.T11969/ValidationMechanism ProfileReference
21.T11969/DataType 21.T11969/ProfileDef
FDO record of attribute definition (PID: 21.T11969/Profile)
