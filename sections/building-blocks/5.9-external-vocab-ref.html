[H5-1g] External Vocabulary Referencing
It is essential for FDOs that existing vocabularies, ontologies, and terminologies can be reused in records (even if they are not in the strict format as described in [H5-1]). This can be carried out by applying external vocabulary referencing.
Figure 6 shows how schema.org terms can be used in an FDO record by applying external vocabulary referencing.
  • The attribute definition 21.T11969/SchemaOrgTerm states that it uses external attribute referencing through the attribute (21.T11969/ValidationMechanism, ExternalVocabularyReference).
  • The syntax definition 21.T11969/SchemaOrgTermSyntax specifies the syntax of the attribute definition, i.e., it must be instantiated by the string “https://schema.org”, which is the schema.org namespace.
  • The FDO 21.123/456 contains the attribute (21.T11969/SchemaOrgTerm, https://schema.org) to indicate that it uses vocabulary from schema.org. Afterwards, any schema.org term can be used as an attribute in the form (<URL of term>, <value>), such as (https://schema.org/priceCurrency, USD).

The detailed requirements for External Vocabulary Referencing are:

Format of an attribute definition that Applies External Vocabulary Referencing:

The following detailed requirements must be fulfilled such that an attribute definition A applies external vocabulary referencing:

  • The attribute definition A MUST explicitly state that it uses this feature by containing the attribute (21.T11969/ValidationMechanism, ExternalVocabularyReference).
  • The attribute definition A MUST specify the namespace of the external vocabulary using the attribute 21.T11969/Data which references a syntax definition that validates the namespace.

Instantiation of External Vocabulary Attributes in the Record:

To use external vocabulary in the FDO record, the record MUST contain
  • an attribute a=(adef,aval) that instantiates attribute definition A, where A applies external vocabulary referencing
  • an attribute b=(bdef,bval) representing a vocabulary term
A and b must fulfill the following requirements:
  • aval is a URL representing a namespace
  • bdef = aval/<term>, i.e., bdef is a URL representing a term under a namespace
  • >bval instantiates bdef.

Validation Rules:

To validate an FDO that contains an attribute a=(adef,aval) applying external vocabulary referencing, it is checked that the namespace aval is valid according to the syntax definition referenced from the attribute definition.
