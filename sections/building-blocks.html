FDO records

The Handle record

Each Handle record contains a list of Handle values. Each Handle value is represented as a JSON object with five types (see also Handle Technical Manual):
  • index (integer)
  • type (string)
  • data (object)
  • ttl (the time-to-live in seconds of the value, an integer (or, in the rare case of an absolute expiration time, that expiration time as an ISO8601-formatted string))
  • timestamp (an ISO8601-formatted string)
The index of Handle values is unique, i.e., it is not possible to have several Handle values with the same index.

The FDO record

The conceptional FDO record contains tuples `(k_1:v_1), (k_2:v_2),…, (k_n:v_n)`.

Mapping the FDO Record to the Handle Record

Each tuple `(k_1:v_1), (k_2:v_2),…, (k_n:v_n)` is assigned a positive integer f_i, where f_1, ..., f_n are mutually different. Assigning integers to tuples should take conventions from Handle identifiers into account (e.g., if `k_i=HS_ADMIN`, then `f_i=100`). Then, the `f_i-th` Handle value is defined by setting:
  • `index:f_i`
  • `type:k_i`
  • `data.value:v_i`
The other types of the `f_i-th` Handle value are usually set according to their default values, but possibly also with different values, if this makes sense in the given context.
  • `data.format:string` (and `data.format=admin` if `k_i=HS_ADMIN`)
  • `ttl:86400`
  • `timestamp:current time`
As an example, the FDO record with handle `123/abcde` containing the three FDO attributes
  • `(0.FDO/Type: 123/45678)`
  • `(0.FDO/Profile: 123/00001)`
  • `(HS_ADMIN: {"handle": "0.NA/21.T11971", "index": 200, "permissions": "111111111111"})`
is mapped to the following Handle record in JSON format:
